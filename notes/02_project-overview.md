## 1. プロジェクト概要

- **アプリ名**: MediQuest（メディクエスト）
- **コンセプト**: 「孤独な国試勉強」を「仲間との冒険」に変える
- **目的**: medilink教材（QB/QA）の進捗を可視化し、友人との共有とゲーミフィケーションによってモチベーションを維持する。
- **ターゲット**: CBT・医師国家試験受験生
- **開発言語**: Python (Django Framework)
- https://docs.google.com/document/d/1uie-79F4gliRMrIZl2J0zzxVusaqp8bkPv5mDBI_hgA/edit?usp=sharing
- https://gemini.google.com/share/5ce4668363f1

## 2. 実装機能要件 (MVP)

### A. ユーザー・認証機能 (Django標準機能)

- [ ] ユーザー登録 / ログイン / ログアウト
- [ ] プロフィール設定（ニックネーム、志望科、現在のレベル）

### B. 記録・データベース機能 (Models)

- [ ] **科目データベース**: 循環器、消化器、呼吸器などを管理
- [ ] **学習ログ入力**:
    - 日付
    - 教材（QB / QA）
    - 科目
    - 解いた問題数 / 視聴時間
- [ ] **進捗保存**: ユーザーごとの累計問題数、累計時間を保存

### C. ゲーミフィケーションロジック (Logic)

- [ ] **経験値(XP)計算**: `問題数 × 10xp` + `時間(分) × 5xp` などで算出
- [ ] **レベルアップ機能**: 累計XPに応じてユーザーレベルが上昇
- [ ] **称号システム**: 「循環器の覇者（循環器1周完了）」などのバッジ付与

### D. 友人共有・SNS機能 (Social)

- [ ] **フレンド登録**: ユーザーID検索で友人を追加
- [ ] **タイムライン**: 友人がログを記録したら「〇〇さんが循環器を50問解きました！」と表示
- [ ] **リアクション**: 「ナイス！」「お疲れ！」ボタンの実装

### E. AI参謀・自動プランニング (Algorithm)

- [ ] **目標設定**: 試験日（国試・CBT）と目標完了日を入力
- [ ] **ノルマ計算**: `(総問題数 - 完了数) ÷ 残り日数` を自動計算し、その日の「クエスト」として提示

## 3. 技術スタック・開発環境

- **Backend**: Python 3.x, Django 5.x
- **Database**: SQLite (開発用) → PostgreSQL (本番用)
- **Frontend**: HTML5, CSS (Bootstrap推奨), JavaScript (Chart.jsなど)
- **Tools**: VS Code, Git/GitHub

# 開発ロードマップと学ぶべき知識

Djangoは「多機能」なフレームワークなので、学ぶ順番を間違えると挫折しやすいです。以下のステップで進めてください。

### STEP 1: Djangoの基本構造 (MVTモデル)

Django特有の「どこに何を書くか」というルールを理解します。

- **学ぶべきこと**:
    - **ProjectとAppの作成**: `startproject`, `startapp` コマンド
    - **MVTパターン**: Model（データ）, View（ロジック）, Template（見た目）の関係性
    - **URL設計**: `urls.py` の書き方（どのURLでどの画面を開くか）

### STEP 2: データベース設計 (Models)

アプリの心臓部です。Excelで表を作るイメージをコードにします。

- **学ぶべきこと**:
    - **ORM (Object-Relational Mapping)**: SQLを書かずにPythonコードでDBを操作する技術。
    - **マイグレーション**: `makemigrations`, `migrate`（DBへの変更反映）
    - **リレーション**: UserモデルとStudyLogモデルをどう紐付けるか（1対多の関係など）

### STEP 3: 画面作成とデータ表示 (Views & Templates)

データを画面に出す作業です。

- **学ぶべきこと**:
    - **Templateタグ**: HTMLの中でPythonの変数を使う方法 (`{{ user.username }}`, `{% for log in logs %}` など)
    - **Bootstrap (CSS)**: デザインを一から作ると大変なので、既存のデザインパーツ集（Bootstrap）を使って、スマホで見やすい画面を作ります。

### STEP 4: フォームとデータ入力 (Forms)

ユーザーが勉強時間を入力する機能です。

- **学ぶべきこと**:
    - **ModelForm**: 定義したモデル（DB）から自動で入力フォームを作るDjangoの便利機能。
    - **バリデーション**: 入力された数字が正しいか（マイナスになっていないか等）のチェック。

### STEP 5: ユーザー認証 (Auth)

- **学ぶべきこと**:
    - **django.contrib.auth**: Djangoには最初から強力なログイン機能がついています。これの使い方を覚えるだけで、セキュリティの高い会員サイトが作れます。

### STEP 6: グラフ描画とプランニング

- **学ぶべきこと**:
    - **Chart.js**: 進捗グラフを表示するためのJavaScriptライブラリ。Djangoからデータを渡してグラフにします。
    - **datetime計算**: 国試までの日数を計算するPythonロジック。
